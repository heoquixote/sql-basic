# 3-2. 데이터 조회 (SELECT ~ FROM)

## 1. 개념 정의

- `SELECT ~ FROM`은 **데이터베이스에 저장된 데이터를 "조회(Read)"하기 위한 가장 기본이 되는 SQL 구문**입니다.
- **"어떤 테이블에서, 어떤 데이터를, 어떻게 가져올 것인가"를 정의하는 구문**입니다.

## 2. 기본 문법 구조

```sql
-- 전체 조회 시
SELECT * FROM 테이블명;

-- 컬럼 선택 시
SELECT 컬럼명 AS 별칭 FROM 테이블명;
```

- `SELECT`: **조회할 컬럼을 지정**
- `FROM`: **데이터를 가져올 테이블을 지정**
- `*`(와일드카드)는 **"모든 컬럼"을 의미**
- 테이블의 전체 데이터를 확인할 때 사용

> ❓ **왜 쿼리 뒤에 `;`를 찍나요?**
>
> `;`는 "여기까지가 하나의 SQL 명령문입니다"라고 DB에게 알려주는 종료 기호입니다.
>
> DB(MySQL)는 우리가 입력한 SQL을 이렇게 해석합니다.
>
> - `;`를 만날 때까지 = 하나의 문장
> - `;`를 만나면 → 그제서야 실행
>
> 즉, DB는 마침표가 찍히기 전까지는 실행하지 않고 계속 기다립니다.

> ❓ **알리아스(Alias)란?**
>
> 조회 결과에서 컬럼명 또는 테이블명에 붙이는 **임시 이름**입니다.
>
> 실제 DB에 저장되는 이름은 바뀌지 않고, **SELECT 결과 화면에서만 적용**됩니다.

> ❓ **DISTINCT란?**
>
> `DISTINCT`는 **SELECT 결과에서 중복된 데이터(값)를 제거하고, 유일한 값만 조회하기 위한 키워드**입니다.
>
> 같은 값이 여러 행에 반복되어 있어도 **하나의 값만 출력**되도록 합니다.
>
> 즉, **"중복 제거 옵션"이며, 데이터 자체를 변경하지 않고 조회 결과만 정제**합니다.

## 3. 문제 풀이

### 1. 사원 테이블을 조회하시오.

- 정답

```sql
SELECT * FROM EMP;
```

### 2. `EMP` 테이블을 조회하여 다음 출력 결과를 만족하는 SQL을 작성하시오.

| 사원번호 | 사원명 | 직급 |
| --- | --- | --- |
| 1000 | 김대표 | CEO |
| 4001 | 최재혁 | 부장 |
| 4002 | 남궁민수 | 대리 |
| 7001 | 백정태 | 상무 |
| 7002 | 김낙수 | 부장 |
| 7003 | 도진우 | 부장 |
| 7004 | 신준섭 | 차장 |
| 7005 | 허태환 | 과장 |
| 7006 | 송익현 | 과장 |
| 7007 | 정성구 | 대리 |
| 7008 | 이명헌 | 대리 |
| 7009 | 권송희 | 사원 |
| 7010 | 채소연 | 사원 |
| 8001 | 이주영 | 작업반장 |

- 정답

```sql
SELECT empno AS `사원번호`
     , ename_ko AS `사원명`
     , joblv AS `직급`
FROM EMP;
```

### 3. `EMP` 테이블에서 **중복을 제외한** 직급 목록을 출력하시오.

| 직급 |
| --- |
| CEO |
| 부장 |
| 대리 |
| 상무 |
| 차장 |
| 과장 |
| 사원 |
| 작업반장 |

- 정답

```sql
SELECT DISTINCT joblv
FROM EMP;
```
