## 학습 목표

- SQL 쿼리가 작성되는 순서와 실행되는 순서를 구분할 수 있습니다.
- 각 절이 어떤 데이터를 대상으로 동작하는지 이해할 수 있습니다.

## 목차

1. 작성 순서와 실행 순서
2. 왜 이 차이가 중요한가

## 1. 작성 순서와 실행 순서

SQL은 보통 `SELECT -> FROM -> WHERE -> GROUP BY -> HAVING -> ORDER BY`처럼 작성하지만,
실제로는 `FROM -> WHERE -> GROUP BY -> HAVING -> SELECT -> ORDER BY` 흐름으로 이해하는 편이 결과를 예측하기 쉽습니다.

## 2. 왜 이 차이가 중요한가

이 순서를 이해하지 못하면 다음처럼 자주 헷갈립니다.

- 왜 `WHERE`에서는 집계 함수를 바로 못 쓰는가
- 왜 `HAVING`이 따로 필요한가
- 왜 `SELECT`에서 만든 별칭을 앞 단계에서 못 쓰는가
