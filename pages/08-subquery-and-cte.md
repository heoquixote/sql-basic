서브쿼리와 CTE를 통해 SQL을 단계적으로 작성하는 방법을 다루는 챕터입니다.

## 1. Sub Query란?

- 서브 쿼리(Sub Query)란 SQL 문장 안에 포함된 또 하나의 `SELECT` 문을 의미합니다.
- 대용량 데이터를 다루다 보면, 컬럼 기반 연산, 사칙연산, 집계, 윈도우 함수, 포맷팅이나 여러 단계의 필터링과 조건 처리가 동시에 필요해집니다.
- 이때 SUB QUERY는 복잡한 연산을 단계별, 구조적으로 나누는 역할을 수행합니다.

## 2. SUB QUERY 실행 흐름

> 가장 안쪽 서브 쿼리 -> 바깥 서브 쿼리 -> 메인 쿼리 순서대로 실행

## 이 챕터의 목표

- 서브쿼리와 CTE의 기본 개념을 설명할 수 있습니다.
- 복잡한 SQL을 더 읽기 쉽게 분해할 수 있습니다.
- 두 방식의 차이와 사용 목적을 비교할 수 있습니다.
