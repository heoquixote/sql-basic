데이터 분석을 하다 보면 결국 "데이터가 어디에 저장되어 있고, 어떤 구조로 연결되어 있는가"를 이해해야 합니다.  
1장은 SQL 문법을 배우기 전에, 데이터베이스를 읽는 기본 눈을 만드는 장입니다.

우리는 먼저 왜 SQL을 배워야 하는지부터 정리합니다.  
현업 데이터는 대부분 여러 테이블이 미리 합쳐진 파일이 아니라 데이터베이스 안에 저장되어 있고, 분석가는 비즈니스 질문을 테이블, 조건, 집계, 기간 관점으로 해석할 수 있어야 합니다.

그 다음에는 데이터베이스의 기본 구조를 배웁니다.  
인스턴스, 데이터베이스, 스키마, 테이블, 컬럼, 행이 각각 무엇을 의미하는지 이해하면, 이후 `SELECT`, `JOIN`, `GROUP BY` 같은 문법이 훨씬 덜 추상적으로 느껴집니다.

이 장에서 특히 익혀야 할 핵심은 다음과 같습니다.

- 데이터 질문을 SQL 질문으로 바꾸는 관점
- 인스턴스, 데이터베이스, 스키마의 차이
- 테이블, 컬럼, 행이 각각 어떤 역할을 하는지
- SQL을 쓰기 전에 어떤 구조를 먼저 확인해야 하는지
