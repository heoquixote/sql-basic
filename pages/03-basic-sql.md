3장은 SQL의 가장 기본이 되는 조회 문법을 배우는 장입니다.  
실무에서 쓰는 대부분의 분석 쿼리는 결국 `SELECT`, `WHERE`, `GROUP BY`, `ORDER BY`를 조합하는 형태로 시작합니다.

가장 먼저 SQL을 사람이 쓰는 순서와 DB가 실제로 처리하는 순서를 구분합니다.  
이 흐름을 이해해야 `WHERE`와 `HAVING`의 차이, 별칭을 어디에서 쓸 수 있는지, 집계 함수가 왜 특정 위치에서만 가능한지를 자연스럽게 이해할 수 있습니다.

그 다음에는 데이터를 가져오는 `SELECT ~ FROM`, 필요한 행만 남기는 `WHERE`, 결과를 정렬하고 개수를 제한하는 `ORDER BY`, `LIMIT`, 그리고 데이터를 묶어 요약하는 `GROUP BY`, `HAVING`으로 이어집니다.

이 장은 단순히 문법을 암기하는 장이 아니라, SQL을 하나의 처리 흐름으로 읽는 감각을 만드는 장입니다.  
즉, "테이블에서 데이터를 가져오고 -> 조건으로 걸러내고 -> 필요하면 그룹으로 묶고 -> 마지막에 보여준다"는 전체 구조를 몸에 익히는 단계라고 보시면 됩니다.

이 장에서 특히 익혀야 할 핵심은 다음과 같습니다.

- SQL 작성 순서와 실행 순서의 차이
- `SELECT ~ FROM`으로 필요한 컬럼을 조회하는 방법
- `WHERE`로 조건을 걸고 비교, 범위, 목록, 패턴 검색을 처리하는 방법
- `ORDER BY`, `LIMIT`으로 결과를 정렬하고 잘라서 보는 방법
- `GROUP BY`, `HAVING`으로 집계와 그룹 조건을 다루는 방법
