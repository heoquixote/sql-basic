# SQL로 배우는 데이터 분석과 실무 쿼리

이 저장소는 `sql-basic` GitHub 리포지토리와 WikiDocs 연동을 전제로 운영하는 SQL 책 프로젝트입니다.
SQL 문법만 외우는 책이 아니라, `왜 이 쿼리가 맞는지`, `왜 여기서 틀리기 쉬운지`, `실무에서는 무엇을 먼저 의심해야 하는지`까지 함께 담을 수 있도록 구성했습니다.

기초 조회부터 조인, 집계, 윈도우 함수, 성능 튜닝, BI/대시보드용 쿼리 설계까지 실무 흐름에 맞춰 정리할 수 있는 책 프로젝트입니다.

## 이 저장소에서 다루는 핵심 주제

- SQL의 실행 순서와 질의 사고방식
- SELECT, WHERE, GROUP BY, HAVING, ORDER BY의 정확한 역할
- JOIN, 서브쿼리, CTE, 집합 연산
- 윈도우 함수와 분석용 쿼리 패턴
- 문자열, 날짜, NULL, CASE 기반 데이터 정제
- 스키마 해석, 데이터 모델링, 그레인(Granularity) 이해
- 성능 최적화와 실행 계획 해석
- BI / 리포팅 / 데이터 분석 실무에 맞는 SQL 작성 방식

## 이런 분께 맞습니다

- SQL을 처음 배우지만 단순 문법이 아니라 실무 기준으로 익히고 싶은 분
- 데이터 분석가, BI 분석가, 데이터 엔지니어, 백엔드 실무자
- JOIN과 집계는 익숙하지만 윈도우 함수와 튜닝이 늘 어렵게 느껴지는 분
- 대시보드용 SQL, 지표 산출 SQL, 데이터 검증 SQL을 체계적으로 정리하고 싶은 분

## 도서 구성

### 01. SQL 사고방식과 실행 순서

SQL은 문장을 위에서 아래로 읽는 언어처럼 보이지만, 실제 실행은 다른 순서로 이뤄집니다.
이 장에서는 `FROM -> WHERE -> GROUP BY -> HAVING -> SELECT -> ORDER BY` 흐름을 중심으로, 왜 초보자가 같은 쿼리를 보고도 다른 결과를 예상하는지부터 정리합니다.

### 02. SQL 실습 환경과 연습 데이터

학습용 환경을 실제로 준비하는 장입니다.
로컬 DB, 클라우드 웨어하우스, 샘플 스키마를 어떻게 고를지와 테이블 관계를 어떤 순서로 읽어야 하는지 다룹니다.

### 03. 기본 조회와 필터링

SELECT, DISTINCT, WHERE, ORDER BY, LIMIT 같은 기본 문법을 다루되, 단순 조회가 아니라 `원하는 행을 정확히 남기는 법`에 집중합니다.
조건 순서, NULL 처리, 문자열 비교, 날짜 필터 같은 실무형 함정을 함께 정리합니다.

### 04. JOIN과 집합 연산

분석 SQL에서 가장 많이 실수하는 영역입니다.
INNER / LEFT / RIGHT / FULL JOIN, UNION / UNION ALL, INTERSECT / EXCEPT를 비교하면서 왜 데이터가 늘어나거나 줄어드는지 원리까지 설명합니다.

### 05. 집계와 지표 계산

COUNT, SUM, AVG, MIN, MAX, GROUP BY, HAVING을 기반으로 KPI와 지표를 만드는 장입니다.
중복 집계, 분모 정의, 비율 계산, 기간 기준 비교처럼 실제 BI 쿼리에서 자주 문제 되는 포인트를 다룹니다.

### 06. 서브쿼리와 CTE

복잡한 질의를 단계별로 쪼개는 방법을 다룹니다.
서브쿼리와 CTE를 각각 언제 쓰는지, 가독성과 재사용성, 최적화 관점에서 어떤 차이가 있는지 함께 정리합니다.

### 07. 윈도우 함수

SQL을 한 단계 끌어올리는 핵심 장입니다.
ROW_NUMBER, RANK, SUM OVER, LAG, LEAD, 이동 평균, 누적값처럼 분석가가 가장 자주 쓰는 패턴을 실무 예제로 풀어갑니다.

### 08. 함수와 데이터 정제

문자열, 숫자, 날짜, CASE, COALESCE, NULLIF 같은 내장 함수를 활용해 데이터를 읽기 좋고 분석하기 좋게 바꾸는 흐름을 다룹니다.
원본 정합성이 완벽하지 않을 때 SQL에서 어디까지 처리할지 판단하는 기준도 포함합니다.

### 09. 스키마 해석과 데이터 모델링

좋은 SQL은 문법보다 스키마 이해에서 시작됩니다.
팩트 테이블, 차원 테이블, 키, 카디널리티, 그레인, 정규화/비정규화 개념을 SQL 관점에서 정리합니다.

### 10. 성능 최적화와 실행 계획

같은 결과를 내는 SQL이라도 비용은 크게 다를 수 있습니다.
인덱스, 파티셔닝, 필터 푸시다운, 불필요한 DISTINCT 제거, 조인 순서, 실행 계획 읽기처럼 성능 문제를 줄이는 기준을 다룹니다.

### 11. 실무 분석 SQL 패턴

리텐션, 코호트, 퍼널, 재구매, 활성 사용자, 이상치 탐지처럼 자주 반복되는 분석 과제를 SQL 패턴으로 정리하는 장입니다.
단순 예제가 아니라 바로 재활용 가능한 구조 중심으로 구성합니다.

### 12. BI / 리포팅 / 데이터 워크플로우에서의 SQL

마지막 장에서는 SQL을 단독 도구가 아니라, 대시보드, ETL/ELT, 지표 정의, 데이터 품질 검증과 연결된 실무 자산으로 다룹니다.
어떤 SQL은 대시보드용으로, 어떤 SQL은 마트 적재용으로 써야 하는지 역할 차이까지 정리합니다.

상세 목차는 [TOC.md](./TOC.md)에서 확인할 수 있습니다.

## 실습 데이터

예제는 아래처럼 데이터 분석과 BI에서 자주 보는 전형적인 비즈니스 스키마를 기준으로 작성할 수 있도록 설계했습니다.

- `orders`: 주문 단위 사실 테이블
- `order_items`: 주문 상세 라인 아이템
- `customers`: 고객 차원
- `products`: 상품 차원
- `events`: 사용자 행동 로그
- `subscriptions`: 구독 및 결제 상태

즉, 전자상거래와 구독형 서비스 양쪽 모두에 재활용 가능한 구조를 기본 실습 모델로 삼습니다.
