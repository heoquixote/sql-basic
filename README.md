# Learners School : SQL 기초

이 저장소는 `sql-basic` GitHub 리포지토리와 WikiDocs 연동을 전제로 운영하는 SQL 입문서 프로젝트입니다.
지금 기준 목차는 `데이터베이스/테이블 이해 -> DBeaver 실습 환경 -> 기초 SQL -> 내장 함수 -> 조건문 -> 윈도우 함수 -> 조인과 유니온 -> Sub Query & CTE` 흐름으로 구성했습니다.

## 도서 구성

### 01. 데이터베이스와 테이블 이해

데이터베이스, 스키마, 테이블, 컬럼, 행 같은 가장 기본적인 구조를 먼저 정리합니다.
SQL 문법을 배우기 전에 “내가 지금 무엇을 조회하는가”를 구조적으로 이해하는 장입니다.

### 02. DBeaver 설치 및 실습 환경 구성

SQL을 직접 실행해 보기 위한 환경을 구성합니다.
DBeaver 설치부터 데이터베이스 연결, 기본 화면 이해, 실습 준비까지 다룹니다.

### 03. 기초 SQL

가장 핵심이 되는 조회 문법을 단계적으로 다룹니다.
쿼리 작성 순서, `SELECT ~ FROM`, `WHERE`, `ORDER BY`, `LIMIT`, `GROUP BY`, `HAVING`까지 SQL의 기본 흐름을 학습합니다.

### 04. 내장 함수

문자열, 숫자, 날짜, 변환, NULL 처리 함수를 통해 데이터를 가공하는 기본기를 익힙니다.
단순 문법 암기보다 “언제 어떤 함수를 써야 하는가”에 초점을 맞춥니다.

### 05. 조건문

`CASE WHEN`을 중심으로 SQL 안에서 조건에 따라 값을 분기하는 방법을 정리합니다.
분류, 라벨링, 구간화 같은 실무 예제에 바로 연결할 수 있는 장입니다.

### 06. 윈도우 함수 (Window Function)

집계는 하되 행을 잃지 않는 SQL의 핵심 개념을 다룹니다.
순위, 누적합, 이전 행 비교 같은 분석용 쿼리의 기초를 잡는 장입니다.

### 07. 조인과 유니온

테이블과 결과 집합을 결합하는 두 가지 핵심 방법을 구분해서 설명합니다.
특히 조인으로 행 수가 왜 늘어나거나 줄어드는지, 유니온은 언제 써야 하는지를 실무 기준으로 정리합니다.

### 08. Sub Query & CTE

복잡한 SQL을 더 읽기 쉽고 단계적으로 만드는 방법을 다룹니다.
서브쿼리와 CTE를 각각 어떤 상황에서 선택할지 비교할 수 있도록 구성합니다.

상세 목차는 [TOC.md](./TOC.md)에서 확인할 수 있습니다.
